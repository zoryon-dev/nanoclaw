## 1. Background & Motivation

Plan 2.5 is live (5 crons, 12-tab workbook, manual + chat-driven flows). It works, but the operator brought a richer source-of-truth document (`Controle_Despesas_Jonas_DOC.md`) that captures three concepts the current planilha cannot express cleanly:

1. **Hierarchical taxonomy.** 3 categories × 13 subcategories (Empresarial / Residencial / Pessoal → IA & LLMs, Saúde, etc). Today `Categorias` is flat and `Lançamentos.categoria` is a single column.
2. **Decision history & lifecycle.** Items that were cut (Chatvolt, GPT Codex), reclassified (Apartamento antigo: Empresarial → Pessoal/Dívidas), renamed (Prestação Carro → Carro Jadiel). Today `Recorrentes.ativo: bool` flattens all of this into a single boolean.
3. **Sensitivity / end-of-term metadata.** Some subcategories (Saúde, Educação, Dívidas) are not "fat to cut" — the agent should treat them with care. Some recorrentes have known end dates (Apartamento antigo → fev/2027). Today neither concept exists in the schema or in Levis's behavior.

Plan 3 extends both the spreadsheet and Levis to make those three concepts first-class. The doc itself becomes a regeneratable artifact (exported on demand from the spreadsheet) — so the design fixes both the model and the medium without locking the operator out of either.

## 3. Spreadsheet schema (Plan 3)

Same workbook ID. **Two new tabs (`Subcategorias`, `Decisoes`).** Existing tabs change as follows.

### 3.1 `Categorias` — now stores only the parent level

| col | type | obs |
|---|---|---|
| `nome` | string | `Empresarial` / `Residencial` / `Pessoal` (3 rows) |
| `escopo` | enum | `PF` | `PJ` | `global` |
| `codigo_prefixo` | string(3) | `EMP` / `RES` / `PES` — used to build `codigo` for Recorrentes |

Migration note: existing rows in this tab (Plan 2.5 categorias flat) are moved to `Subcategorias` and the table is reseeded with the 3 parent rows.

### 3.2 `Subcategorias` (new) — child level

| col | type | obs |
|---|---|---|
| `nome` | string | `IA & LLMs`, `Saúde`, `Moradia`, etc. (13 rows initially) |
| `categoria_pai` | string | FK to `Categorias.nome` |
| `escopo` | enum | `PF` | `PJ` | `global` |
| `codigo_prefixo` | string(3) | `IAL`, `SAU`, `MOR`, etc. — used to build `codigo` (full format `{CAT_PREFIXO}-{SUBCAT_PREFIXO}-{NNN}`) |
| `sensibilidade` | enum | `alta` | `media` | `nenhuma` |
| `nao_sugerir_corte` | bool | true for Saúde, Educação, Dívidas |

Seed values come from §5.

### 3.3 `Recorrentes` — extended

| col | source | obs |
|---|---|---|
| `id` | existing | unchanged (`rec-XXXXXX` technical) — FK target for Lançamentos.recorrente_id |
| `escopo` | existing | unchanged |
| `nome`, `tipo`, `valor` | existing | unchanged |
| `categoria` | existing | now points to parent (`Empresarial`) |
| **`subcategoria`** | **new** | FK to `Subcategorias.nome` |
| **`codigo`** | **new** | format `{Categoria.codigo_prefixo}-{Subcategoria.codigo_prefixo}-{NNN}`, e.g. `EMP-IAL-001`. Generated by Levis on cadastro by looking up the two prefixos and `NNN = max(seq existente nesse prefixo composto) + 1`, zero-padded to 3 digits. Immutable after creation |
| `frequencia`, `dia_do_mes`, `proxima_data`, `pago_no_mes` | existing | unchanged |
| **`status`** | **new** | enum `ATIVO` | `CORTADO` | `PENDENTE` | `ENCERRADO` |
| **`data_corte`** | **new** | date — NULL when status=ATIVO |
| **`motivo_corte`** | **new** | text — free-form, NULL when status=ATIVO |
| **`termina_em`** | **new** | date — NULL if no end date known |
| **`parcelas_restantes`** | **new** | int — NULL if no end date / open-ended |
| ~~`ativo: bool`~~ | **REMOVED** | replaced by `status` (migration: TRUE → ATIVO, FALSE → CORTADO with motivo="(legado pre-Plan-3)") |

**Status semantics:**
- `ATIVO`: in active billing
- `CORTADO`: cancelled — preserved for history with `data_corte` + `motivo_corte`
- `PENDENTE`: planned, not yet in billing (e.g. 4ª D-API)
- `ENCERRADO`: end date reached naturally (e.g. Apartamento antigo após fev/2027). Set by the monthly cron when `termina_em <= today`.

### 3.4 `Lançamentos-PF` / `Lançamentos-PJ` — extended

Adds one column:
| col | obs |
|---|---|
| `subcategoria` | FK to `Subcategorias.nome` |

Existing rows leave `subcategoria` blank. Levis fills on next touch ("editei o último") or leaves blank for old data — backfill is **out of scope** for Plan 3.

### 3.5 `Decisoes` (new) — timeline of structural changes

| col | type | obs |
|---|---|---|
| `data` | date | when the decision was made |
| `item_id` | string | `codigo` from Recorrentes (e.g. `EMP-IAL-001`) — NULL when the decision is structural (e.g. "renamed subcategoria") |
| `tipo` | enum | `corte` | `reclassificacao` | `adicao` | `correcao` | `renomeacao` | `migracao` |
| `detalhes` | text | one-line summary |
| `impacto_mensal` | number | R$ delta to monthly budget (signed; negative = saved) |

Initial seed: 10 rows from §7 of `Controle_Despesas_Jonas_DOC.md` + 1 row `tipo=migracao` at the end of the bootstrap.

### 3.6 Tabs that do NOT change

`Dashboard`, `Orçamento`, `Projeção`, `Lembretes`, `Contas`, `MeiosPagamento`, `Recebiveis`, `_Log` — unchanged. Dashboard formulas may later be updated to aggregate by `subcategoria` (out of scope for Plan 3 PR 1).

### 3.7 Concepts that intentionally do NOT get a tab

- **Calendar by day** — Levis generates dynamically from `Recorrentes` grouping by `dia_do_mes`. No materialization tab.
- **Risks to monitor** — narrative content. Stays in the markdown doc only; Levis reads when relevant.

## 4. Levis behavior (Plan 3)

`groups/finance/system-prompt.md` + `groups/finance/CLAUDE.md` updates.

### 4.1 New intents

| Intent | Triggers | Behavior |
|---|---|---|
| `exportar_doc` | "exporta o doc", "atualiza o markdown", "regenera o doc canônico" | See §7 |
| `cortar_recorrente` | "corta o X", "cancela o X", "X foi cancelado" | Card → `status=CORTADO`, ask for `motivo_corte`, `data_corte=hoje`, +1 row in `Decisoes` (`tipo=corte`, `impacto_mensal=-valor`) |
| `auditar_assinaturas`, `revisao_estrutural`, `revisao_anual` | **only via cron** (§6); user can force ("audita as assinaturas") | Each runs the prompt defined in the cron's job file |

### 4.2 Updated intents

- **`registrar_despesa` / `registrar_receita`** — card now includes `subcategoria` (validated against `Subcategorias` for the chosen escopo). If ambiguous, Levis asks 1 question for subcategoria. Lançamento row written with both `categoria` and `subcategoria`.
- **`cadastrar_recorrente`** — card adds:
  - `subcategoria` (required, validated)
  - `termina_em` (optional — pergunta "tem data de fim?")
  - `parcelas_restantes` (optional, only if `termina_em` set)
  - `codigo` (auto-suggested — Levis computes next `{CAT}-{SUBCAT}-{NNN}` sequence; user can accept or edit)
- **`sugerir_economias`** — **rule change**: before any cut suggestion, Levis must filter out `Subcategorias.nao_sugerir_corte = TRUE`. If little remains, say so explicitly ("o orçamento já está enxuto — só posso sugerir cortes em [lista]; saúde/educação/dívidas com prazo são intocáveis sem orientação sua").

### 4.3 New principles in `system-prompt.md`

1. **Tom em categorias sensíveis** — Saúde, Educação, Dívidas com prazo, Alimentação são **nunca** chamadas de "gordura", "candidato a corte", "supérfluo". Levis: "Saúde é categoria sensível — só sugiro mexer se você trouxer pra mim, não inicio."
2. **Hierarquia de classificação** — toda nova despesa: 1º cat, 2º subcat, 3º código (se virar recorrente). Se user dá só descrição, Levis sugere cat+subcat baseado em §2.4 do doc canônico ("Regras de classificação para itens novos").
3. **Status histórico** — itens com `status=CORTADO` ou `ENCERRADO` **nunca são deletados** de `Recorrentes`. Lookups operacionais (consulta, dashboard, finance-sweep) filtram `status=ATIVO`; consultas históricas ("o que eu cortei?") incluem CORTADO/ENCERRADO.
4. **Doc canônico read-on-need** — Levis lê `groups/finance/Controle_Despesas_Jonas_DOC.md` via `Read` quando o contexto exige (sensibilidade desconhecida, decisão histórica, risco específico). Não carrega na inicialização.

### 4.4 `CLAUDE.md` updates

- Schema crítico section reflects new columns / new tabs.
- New section **"Categorias sensíveis"** with 1 line + pointer to doc §8.4.
- New section **"Doc canônico"** linking to `Controle_Despesas_Jonas_DOC.md` with instructions:
  > Read it when you need: classification rules (§2.4), historical decisions (§7), end-of-term schedule (§6), sensitive categories (§8.4), or risks (§8.3). Don't load it at session start — it's reference, not context.

### 5.6 Idempotency rules

Every step is idempotent:
- Schema: `if column exists, skip` per column.
- Tabs: `if tab exists, skip add`.
- Taxonomia seed: `lookup by nome, if exists skip`.
- Recorrentes: `lookup by codigo, if exists skip`.
- Decisoes: not idempotent (timeline append) — `if Decisoes already has row 'Plan 2.5 → Plan 3 bootstrap complete'`, the whole Step E is skipped.

Re-running the prompt on an already-migrated workbook is a no-op.

## 7. Intent `exportar_doc`

**Triggers:** "exporta o doc", "atualiza o markdown", "regenera o doc canônico", "atualiza o controle de despesas".

**Workflow:**

1. **Read aggregation:**
   - `GOOGLESHEETS_VALUES_GET` on `Categorias`, `Subcategorias`, `Recorrentes` (all status), `Decisoes`.
2. **Aggregate in memory:**
   - Total mensal (sum of `valor` where `status=ATIVO`)
   - Distribuição por `categoria` (3 buckets)
   - Top 5 individuais
   - Inventário completo (33 items, grouped by `categoria` > `subcategoria`)
   - Itens CORTADOS (filter `status=CORTADO`)
   - Calendário por `dia_do_mes` (group sum + flag dias críticos >R$ 4.000)
   - Compromissos com `termina_em IS NOT NULL`
3. **Render markdown** following the structure of the current `Controle_Despesas_Jonas_DOC.md` (the file itself becomes the canonical template). Bump version number and `Última atualização` date.
4. **Compute diff vs existing doc:** read current `groups/finance/Controle_Despesas_Jonas_DOC.md`, generate one-line summary of changes ("3 itens adicionados, 1 cortado, valor total -R$ 35").
5. **Confirmation card:**
   ```
   📝 Atualizar Controle de Despesas?
   📄 Atual: N linhas, vN.M, atualizado em DD/MM
   📄 Novo: M linhas, v(N+0.1).0, hoje
   📊 Diferenças: {diff resumo}
   [✓ Sim]  [❌ Cancelar]
   ```
6. **On confirm:** write to `/workspace/agent/Controle_Despesas_Jonas_DOC.md` (Levis's writable workspace, which is the host's `groups/finance/Controle_Despesas_Jonas_DOC.md`).
7. **Hint operator:** "Doc atualizado em `groups/finance/Controle_Despesas_Jonas_DOC.md`. Commita com `git add groups/finance/Controle_Despesas_Jonas_DOC.md && git commit -m 'chore(finance): regen doc'`."

**Where the doc lives:**

- Repo root currently holds `Controle_Despesas_Jonas_DOC.md` (the v2.0 hand-written input). Plan 3 moves it to `groups/finance/Controle_Despesas_Jonas_DOC.md` so it's inside the Finance agent's workspace and Levis can read it via `Read` from `/workspace/agent/`. The move is a one-time operation in PR 1.

**`_Log` row:** every successful export writes a row `(timestamp, "exportar-doc", "success", lines_count, "")` so audit is preserved.

- **Cambial reconciliation** — bootstrap uses doc's R$ 5,40 USD rate as static; recalculation when rate moves is out of scope.
- **`Riscos` tab** — riscos stay in the markdown doc (§8.3). Promoting to structured tab is a Plan 3.1 candidate.
- **Doc as `@./Controle_Despesas_Jonas_DOC.md` import in CLAUDE.md** — explicitly NOT done in Plan 3 (token cost). Levis reads on demand.

## 10. Risks & validation

| Risk | Likelihood | Mitigation |
|---|---|---|
| Migration fails partway, planilha left half-reformed | low (migration is idempotent — re-run resumes) | Each step verifies before-after counts; Levis emits error message on failure with which step failed |
| Levis system-prompt regression breaks existing intents | medium | PR 2 smoke checklist covers `registrar_despesa`, `consulta`, `marcar_pago`, `desfazer` (the 4 most-used) |
| `exportar_doc` overwrites a manually edited doc | medium | Confirmation card shows line counts and diff summary; Jonas can decline |
| Crons fire on the wrong day due to BRT/UTC confusion | low | Same cron-expr-parser used by Plan 2.5 (`cron-parser` with `tz: 'America/Sao_Paulo'`); test in unregister/register flow |
| `codigo` collision when Levis auto-suggests | low | Sequence number = max(existing) + 1, scoped per `{CAT}-{SUBCAT}`; verified by lookup before insert |
